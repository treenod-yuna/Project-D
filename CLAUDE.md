# CLAUDE.md - 매치3 기믹 프로토타이핑 플랫폼

## 1. 프로젝트 개요
**기획자가 웹 브라우저에서 자연어로 기믹을 설명하면, 실제 플레이 가능한 매치3 보드 위에 즉시 생성되어 기믹 간 상호작용까지 검증할 수 있는 도구**

- 웹 브라우저에서 바로 실행 (HTML/Canvas/JS)
- 로얄매치/매치빌런 스타일 매치3 코어 엔진
- 기믹 프레임워크: 다중 기믹 공존 및 상호작용 검증
- AI 기믹 생성: 자연어 → Gemini API → 기믹 자동 생성 (추후 Claude API로 교체 가능)
- 기획자 리소스(이미지) 업로드 → 기믹 외형으로 적용
- 실시간 파라미터 제어 (수치 입력, 이전/현재 값 비교)
- 가독성, 유지보수성, 확장성을 최우선 가치로 합니다
- 코드는 명확하고 단순하게 작성합니다

### 핵심 목표
- **빠른 피드백 루프**: 기획자가 아이디어 입력 → 30초 안에 플레이 가능
- **기믹 상호작용 검증**: A기믹과 B기믹이 같은 보드에서 충돌/공존 테스트
- **코딩 없이 기획자 단독 사용 가능**

### 현재 범위에 없는 것 (추후 확장 가능)
- 보드 크기 제어
- 스테이지/난이도 곡선 설계
- 클리어 조건 처리
- 밸런스 시뮬레이션
- Unity 저장/공유/내보내기 (확장성만 열어둠)
- Unity에서 기믹별 이펙트/연출 연결 (기믹 JSON에 effectRef 필드로 확장 가능하게 설계)

## 3. 시스템 아키텍처

### 전체 구조
```
[3매치 코어 엔진]
  - 보드 생성, 스왑, 매치 감지, 제거, 낙하, 리필, 연쇄, 셔플
  - 특수 블록 (4매치→로켓, 5매치→레인보우, L/T→폭탄)
  - 특수 블록 간 조합 효과
       ↓
[기믹 프레임워크]
  - 기믹 생명주기 (생성 → 발동 조건 → 효과 → 파괴)
  - 기믹 유형: 레이어형, 확산형, 수집형, 발동형, 방해형
  - 다중 기믹 공존 및 상호작용 규칙
  - 연쇄 트리거 처리
       ↓
[AI 기믹 생성기]
  - 자연어 입력 → Gemini API → 기믹 설정 JSON
  - API 모듈 분리 (Gemini → Claude 교체 용이하게)
       ↓
[플레이 가능한 미리보기]
  - 기획자가 직접 플레이하며 상호작용 확인
  - 실시간 파라미터 제어
```

### 기믹 유형 분류
| 유형 | 예시 | 동작 패턴 |
|------|------|-----------|
| 레이어형 | 얼음, 체인 | N번 인접 매치로 단계별 파괴 |
| 확산형 | 꿀, 잡초 | 턴마다 인접 칸으로 퍼짐 |
| 수집형 | 아이템 낙하 | 보드 아래로 이동시키면 수집 |
| 발동형 | 폭탄, 로켓 | 매치 시 범위 효과 발동 |
| 방해형 | 상자, 돌 | 매치 불가, 특정 조건으로만 제거 |

## 7. 기믹 프레임워크 규칙

### 기믹 생명주기
```
생성 → 보드 배치 → 조건 감지 → 효과 발동 → 상태 변경/파괴
```

### 기믹 정의 구조
```json
{
  "id": "ice_block",
  "name": "얼음 블록",
  "type": "layer",
  "size": [1, 1],
  "hp": 2,
  "triggerCondition": "adjacentMatch",
  "effect": "reduceHp",
  "visual": {
    "image": "ice_block.png",
    "fallbackColor": "#A0D2FF",
    "fallbackIcon": "❄️"
  }
}
```

### 기믹 사이즈
- 기믹은 1x1, 2x2, 1x2, 2x1 등 다양한 크기를 가질 수 있음
- size: [가로, 세로] 칸 수로 정의
- 2x2 이상 기믹은 여러 칸을 차지하며, 해당 칸들의 매치/낙하/스왑 처리에 영향
- 큰 기믹이 차지하는 칸에는 일반 블록 배치 불가
- 큰 기믹의 파괴 조건: 차지하는 칸 중 하나라도 인접 매치 발생 시 (기본값, 기믹별 커스텀 가능)

### 기믹 상호작용 규칙
- 기믹 간 우선순위 체계 필요
- 연쇄 트리거: A 기믹 발동 → B 기믹에 영향 → B 기믹 발동 가능
- 충돌 시 명확한 규칙 정의 (어떤 기믹이 먼저 처리되는지)
- 새 기믹 추가 시 기존 기믹과의 상호작용 자동 테스트 가능해야 함

## 10. AI 기믹 생성 규칙

### 입력 방식
- 자연어 텍스트 입력 (Gemini API로 해석)
- 예: "폭탄 블록 - 매칭되면 주변 2칸을 모두 제거"
- 예: "얼음 블록 - 2번 매칭되어야 파괴됨, 반투명 파란색"

### 리소스 적용
- 기획자가 이미지 파일(PNG/SVG)을 업로드하면 기믹 외형으로 사용
- 리소스 없으면 기본 도형 + 색상 + 텍스트/이모지로 표시
- 파일명과 기믹 특성을 연결 (예: "ice_block.png은 얼음 기믹")

### 기믹 저장 기능
- 자연어로 생성 → 플레이 테스트 → "저장" 버튼 클릭 시 Block_Type JSON에 자동 추가
- AI가 자연어를 파싱한 결과(속성값)를 그대로 Block_Type 형식으로 저장
- ID 자동 채번, 나머지 속성은 AI 해석 결과에서 자동 매핑
- 저장 후 기획자가 수동으로 미세 조정 가능

### API 모듈 설계
- LLM API 호출 부분을 독립 모듈로 분리
- 현재: Gemini 무료 API
- 추후: Claude API로 한 파일만 수정하여 교체 가능
- API 실패 시 규칙 기반 파싱으로 폴백
