# Explore 에이전트 정의

## 역할
코드 작성 전 리서치 전담. 레퍼런스 코드 분석, API 문서 확인, 패턴 조사 등을 수행한다.

## 병렬 실행
여러 주제를 동시에 조사할 때 복수의 Explore 에이전트를 병렬로 실행 가능하다.

## 호출 시점
- 구현 전 레퍼런스 구현체 분석이 필요할 때
- 외부 API 스펙/제한 사항 확인이 필요할 때
- 엣지케이스 패턴을 수집할 때
- 기술적 의사결정에 근거 자료가 필요할 때

## 이 프로젝트에서의 용도
- 오픈소스 매치3 구현체 분석 및 엣지케이스 패턴 수집
- 로얄매치 스타일 특수 블록 조합 규칙 조사
- Gemini 무료 API 연동 방법/제한 확인
- Canvas 애니메이션 패턴 조사

## 호출 예시

### 매치3 레퍼런스 조사
```
Task(subagent_type="Explore")
"웹 기반 매치3 오픈소스 구현체를 조사해줘.
- 보드 데이터 구조, 매치 감지 알고리즘
- T/L 매치 처리, 특수 블록 생성/조합
- 낙하/연쇄 처리, 셔플/데드락 감지
- 코드 작성하지 말고 리서치만 수행"
```

### API 문서 확인
```
Task(subagent_type="Explore")
"Gemini 무료 API의 현재 무료 티어 제한을 조사해줘.
- 사용 가능한 모델, RPM/일당 요청 제한
- REST 엔드포인트, 인증 방식
- CORS 이슈, 웹 클라이언트 직접 호출 가능 여부"
```

### 병렬 실행 예시
동시에 3개 에이전트를 띄워 각각 다른 주제 조사:
- 에이전트 1: 매치3 코어 로직 레퍼런스
- 에이전트 2: 기믹 시스템 패턴
- 에이전트 3: Gemini API 연동 방법

## 주의사항
- 코드 작성은 하지 않는다 (리서치 전용)
- 결과는 구조화된 형태로 반환받는다
- 불확실한 정보는 출처와 함께 표기한다
