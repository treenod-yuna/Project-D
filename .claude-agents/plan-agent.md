# Plan 에이전트 정의

## 역할
아키텍처 설계 전담. 리서치 결과를 바탕으로 파일 구조, 모듈 분리, 인터페이스 설계 등 구현 계획을 수립한다.

## 호출 시점
- Explore 에이전트의 리서치 결과가 모인 후
- 새로운 모듈/시스템 설계가 필요할 때
- 기존 구조에 큰 변경이 필요할 때

## 이 프로젝트에서의 용도
- 기믹 프레임워크 아키텍처 설계
- 파일 구조, 모듈 분리 계획
- 이벤트 시스템 설계
- 데이터 구조 설계 (보드, 타일, 기믹 등)

## 호출 예시

### 기믹 프레임워크 설계
```
Task(subagent_type="Plan")
"매치3 기믹 프로토타이핑 플랫폼의 상세 구현 계획을 설계해줘.

리서치 결과 요약: [Explore 결과 전달]

설계할 내용:
- 파일 구조 (구체적 파일명과 역할)
- 구현 순서 (의존성 기반, 단계별)
- 핵심 클래스/함수 설계
- 데이터 구조 (보드, 타일, 기믹, 파라미터)
- 이벤트 시스템
- 기믹 프레임워크 인터페이스
- 코드 작성하지 말고 설계만 수행"
```

### 모듈 리팩토링 설계
```
Task(subagent_type="Plan")
"현재 구조에서 [특정 모듈]을 분리/개선하는 방안을 설계해줘.
- 현재 문제점
- 변경 후 구조
- 영향받는 파일 목록
- 마이그레이션 순서"
```

## 주의사항
- 코드 작성은 하지 않는다 (설계 전용)
- Explore 에이전트의 리서치 결과를 입력으로 받는다
- 설계 결과는 구현 단계에서 바로 사용할 수 있을 정도로 구체적이어야 한다
