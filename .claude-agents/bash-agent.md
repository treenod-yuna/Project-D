# Bash 에이전트 정의

## 역할
명령어 실행 전담. 로컬 서버 실행, 테스트 스크립트 실행, 빌드/검증 작업을 수행한다.

## 호출 시점
- 구현한 코드를 브라우저에서 테스트할 때
- 매치 판정 로직 단위 테스트 실행할 때
- 엣지케이스 자동 검증이 필요할 때
- 프로젝트 초기 세팅 (폴더 생성 등)

## 이 프로젝트에서의 용도
- 로컬 HTTP 서버 실행 (브라우저 테스트)
- Node.js 기반 매치 판정 로직 단위 테스트
- 엣지케이스 자동 검증 스크립트 실행

## 호출 예시

### 로컬 서버 실행
```
Task(subagent_type="Bash")
"프로젝트 루트에서 로컬 HTTP 서버를 실행해줘.
- npx http-server 또는 python -m http.server 사용
- 포트: 8080
- 브라우저에서 접근 가능한지 확인"
```

### 단위 테스트 실행
```
Task(subagent_type="Bash")
"매치 감지 로직 단위 테스트를 실행해줘.
- node tests/match.test.js 실행
- 실패한 케이스가 있으면 상세 내용 보고"
```

### 엣지케이스 검증
```
Task(subagent_type="Bash")
"다음 엣지케이스 검증 스크립트를 실행해줘.
- T/L 모양 동시 매치 판정
- 연쇄 중 특수 블록 생성 타이밍
- 셔플 무한루프 방지
- 결과를 성공/실패로 정리해서 보고"
```

## 주의사항
- 파일 읽기/쓰기는 전용 도구(Read/Write/Edit)를 사용한다
- Bash는 명령어 실행에만 사용한다
- 장시간 실행되는 작업은 백그라운드로 실행 가능하다
