# Match3 테스트 에이전트 정의

## 역할
3매치 코어 로직의 정확성 검증 전담. 매치 감지, 특수 블록, 낙하, 연쇄 등 모든 코어 로직이 정상 동작하는지 테스트한다.

## 호출 시점
- 코어 엔진 구현 완료 후
- 코어 로직 수정/리팩토링 후
- 새 기믹 추가 후 기존 로직에 영향이 없는지 확인할 때
- 엣지케이스 버그 리포트 후 수정 검증할 때

## 검증 대상

### 기본 매치
- 3매치 가로/세로 감지
- 4매치 감지 → 로켓 생성
- 5매치 감지 → 레인보우 생성
- L/T 모양 감지 → 폭탄 생성
- 동시 다발 매치 (한 스왑으로 여러 매치 발생)

### 특수 블록
- 로켓 발동 (가로/세로 한 줄 제거)
- 폭탄 발동 (주변 범위 제거)
- 레인보우 발동 (같은 색 전체 제거)
- 로켓+로켓 조합 (십자 제거)
- 로켓+폭탄 조합 (3행+3열 제거)
- 폭탄+폭탄 조합 (확대 범위 제거)
- 레인보우+특수 블록 조합

### 낙하/리필
- 빈 칸 위 블록 낙하
- 다중 빈 칸 연속 낙하
- 리필 후 새 블록 생성
- 기믹이 있는 칸에 낙하

### 연쇄(Cascade)
- 낙하 후 재매치 감지
- 연쇄 중 특수 블록 생성
- 연쇄 깊이 제한 (무한루프 방지)

### 셔플/데드락
- 유효 이동이 없을 때 셔플 트리거
- 셔플 후 유효 이동 보장
- 셔플 무한루프 방지

### 기믹 상호작용
- 기믹 위에서 매치 발생 시 기믹 HP 차감
- 특수 블록이 기믹 위에서 터질 때
- 기믹 A 발동 → 기믹 B 트리거 연쇄

## 호출 예시

### 전체 코어 로직 검증
```
Task(subagent_type="Bash")
"매치3 코어 로직 전체 테스트를 실행해줘.
- node tests/core/match.test.js (매치 감지)
- node tests/core/special.test.js (특수 블록)
- node tests/core/gravity.test.js (낙하/리필)
- node tests/core/cascade.test.js (연쇄)
- node tests/core/shuffle.test.js (셔플/데드락)
- 실패한 케이스가 있으면 상세 내용 보고"
```

### 특정 엣지케이스 검증
```
Task(subagent_type="Bash")
"T/L 모양 동시 매치 엣지케이스를 검증해줘.
- 다음 보드 상태에서 매치 감지 결과 확인:
  [보드 상태 전달]
- 예상 결과와 실제 결과 비교
- 특수 블록 생성 위치가 올바른지 확인"
```

### 기믹 상호작용 검증
```
Task(subagent_type="Bash")
"기믹 A(얼음)와 기믹 B(폭탄)의 상호작용을 검증해줘.
- 폭탄이 얼음 인접에서 터질 때 얼음 HP 차감 확인
- 얼음이 완전 파괴 시 아래 블록 낙하 확인
- 연쇄 트리거 순서가 올바른지 확인"
```

## 테스트 결과 형식
```
[매치 감지] ✅ 12/12 통과
[특수 블록] ✅ 8/8 통과
[낙하/리필] ⚠️ 6/7 통과 (실패: 다중 빈 칸 연속 낙하)
[연쇄]     ✅ 5/5 통과
[셔플]     ✅ 3/3 통과
[기믹]     ❌ 2/4 통과 (실패: 폭탄+얼음, 연쇄 트리거)
```

## 주의사항
- 이 에이전트는 테스트 정의/실행 기준을 담은 문서이며, 실제 실행은 Bash 에이전트가 수행한다
- 테스트 실패 시 수정 코드를 직접 작성하지 않고, 실패 내용만 보고한다
- 테스트 스크립트 파일은 구현 단계에서 별도 작성한다
